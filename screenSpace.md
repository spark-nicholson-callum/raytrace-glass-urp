# スクリーンスペースレイトレーシング

レイトレの一番上りやすい段階です。

レイトレの一番難しいポイントが当たったメッシュの色を計算することだと思いますね。
ちょっと意外かもしれないが、GPU がその情報を全く管理しません。
単純に当たったメッシュの ID しか管理していない。

つまり、テクスチャの参考、ライティング、影、ラスターの描画処理などのすべてがレイトレ用のシェーダーに再現しないといけません。

それはさすがに大変ですので、まずはすでにある色情報を使うことが楽です。
今まで描画したものを参考してガラスを描画する。

もちろん屈折、と特に反射、だとこれはすべての色を習得できるわけございません。
反射でカメラの後ろや、屈折で物の後ろなど、画面外の描画も必要になりますが、とりあえず簡単なやり方で。

## レイの仕組み
今回はすごく簡単なレイの仕組みで行きます。
レイは個体2ステップにします。

1. ガラスの中通って、ガラスの後ろ側に当たります
2. 屈折してガラスから出て世界の何かに当たる。

結果が3パターンがあります

### ガラスから出るときの全反射
屈折ができないケース、リアルなガラスの描画にとっても大事なケースですが、今回は無視します。
実装中の時は全反射のケースは固定の色を出力して後回ししましたね。

ちなみに完璧な球体ならこのケースが必ずないですよ。

### 外のレイが何も当たらない
一番簡単なケースです、何も当たってないのでスカイボックスを描画するべき。
これは単純にシェーダーにスカイボックステクスチャの参考を持ち運んでレイの方向で参照すれば終わりです。

### 外のレイがメッシュに当たった
ここですね。
まず、私たちがレイがどこに当たるのワールド座標の情報があります。
ほしい情報がそのワールド座標に相当するスクリーン座標に変更が必要。
Unity がクリップスペースまでの変換が用意してくれますが、それかちょっと自分で変換必要です

```hlsl
float4 clipPos = mul(_ViewProjMatrix, float4(pos, 1.0));
float2 ndc = clipPos.xy / clipPos.w;
ndc.y = -ndc.y;
float2 hitUv = ndc * 0.5 + 0.5;
```
これを使ってスクリーンのテクスチャを参考して色情報を取得します

ですが、これはレイ当たった場所ではないかもしれないです！
画面に描画したものの後ろ可能性もあります。

カメラのデプステクスチャも参考して、ワールド座標に戻って同じ場所かどうか確認できます。
