# ハイブリッドレイトレーシング

現在のハードレイトレーシングがかなり季節的な技術ですよね、昔のレイトレーサーに比べると何百回早くなっています。
ですが、ゲームみたいなリアルタイムのアプリにて、全画面のモンテカルロ法のパストレーシングがまだ不可能です。
どこまで技術が進んでもラスター描画の速さに勝てません、今もおそらくこれからもハイブリッドの描画が定番かと思います。
基本はラスター描画に任せて、必要な所だけレイトレーシングを利用します。

「パストレーシング」を利用するゲームがあちこち存在しますが、それでも普通ラスター描画とAIディノイザーを利用してパストレーシングのもどきを作っています。
結果は本格パストレーシングと近いが、これもあくまでハイブリッドなアプローチだと思います。
開発者がどこがレイトレが必要を選ぶ感じではなく、すごい数学を利用して画面全体にどんなレイが一番大事を自動に選んで、残りのデータがAIに任せるって感じですね。

今回はその魔法の世界ではなく、単純にガラス以外のデータが普通のラスター描画で行って、ガラスの反射と屈折はレイトレーシングで描画する形にします。

## コンパクション
普通のレイトレーシングシェーダーだとフラグメントシェーダーと同じ2次元のスレッドグループを利用して計算します。
基本的に問題ないが、各グループにレイトレーシングが不要なピクセルが含まれているケースがあるため無駄な処理を行っています。

最適化が簡単です、必要なピクセルを事前に取得し、1次元の配列に保存し、1次元のスレッドグループで処理を行います。
基本的に3フェーズで計算します

1. 普通のフラグメントシェーダーで必要なピクセルと不要なピクセルを判別してバッファーに書き込む。
2. コンピュートシェーダーで必要なピクセルの情報を AppendBuffer に保存しておく
3. 1次元のスレッドグループで AppendBuffer のデータを読み込んで、レイを出す

普通のレイトレーシングは 1. と 2. が必要ないが、その変わりに重いレイトレーシングの処理が完全に必要な所のみ飛ばしています。

## 第1レイの最適化
定番な考え方だとカメラからレイを飛ばして当たったメッシュの情報を使って次のレイを飛ばします。
ですが、この最初のレイがどこに当たるがすでに分かります、グラスに当たるところしかレイ飛ばしていませんので。

コンパクションの時はピクセルの場所だけではなく、ノーマルとデプスも保存したら直接2番目のレイの初期位置と方向が計算できます。

## プロジェクション
コンピュートシェーダーで頑張ってレイトレーシングを利用してレンズの描画情報が計算できていますが、その情報を画面に描画する必要がまだあります。
これはプロジェクションっといいます。

これはグラスのメッシュが2回目に描画して、コンピュートシェーダーで計算したデータを画面に描画する。
とても当たら前のこと言ってる気がしますがが、しっかり順番を守らないといけません。

まあ、ここでは単純に色を画面に出すだけではなく、鏡面反射光なでの計算も追加でよりリアルな感じできます。
